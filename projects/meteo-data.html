IoT · BUT R&T

Analyse de données météorologiques.

Système IoT complet de collecte, stockage, analyse et visualisation de données météo — capteurs ESP32, MQTT, InfluxDB et tableaux de bord Grafana.

Type
Académique – BUT R&T
Durée
6 semaines
Rôle
Analyste de données
Année
2022
01

Objectif du projet

Ce projet consistait à concevoir et mettre en œuvre un système complet de collecte, stockage, analyse et visualisation de données météorologiques. L'objectif était de créer un réseau de capteurs IoT capable de collecter des données environnementales (température, humidité, pression atmosphérique, vitesse du vent, etc.), de les transmettre à une base de données centralisée, puis d'analyser ces données pour en extraire des tendances et des corrélations significatives.

Le projet s'inscrivait dans une démarche d'exploration des technologies IoT appliquées à la surveillance environnementale, avec un focus particulier sur l'optimisation des protocoles de communication pour les dispositifs à faible consommation et la mise en œuvre de techniques d'analyse de données pour identifier des modèles météorologiques locaux.

02

Architecture du système

  • Capteurs météorologiques basés sur des modules ESP32 avec divers capteurs environnementaux
  • Protocole MQTT pour la communication efficace entre les capteurs et le serveur central
  • Broker MQTT (Mosquitto) pour la gestion des messages entre les différents composants
  • Base de données Time Series (InfluxDB) optimisée pour le stockage de données temporelles
  • Système d'analyse en temps réel pour le traitement des données entrantes
  • Scripts d'analyse Python utilisant pandas et NumPy pour l'analyse statistique
  • Interface de visualisation (Grafana) pour présenter les données et tendances sous forme de tableaux de bord interactifs
  • Système d'alerte pour notifier des conditions météorologiques exceptionnelles
03

Défis rencontrés

  • Conception de stations météo autonomes fonctionnant sur batterie avec une autonomie suffisante
  • Optimisation de la transmission des données pour minimiser la consommation énergétique
  • Gestion de la connectivité intermittente des capteurs et récupération des données manquantes
  • Calibration des capteurs pour garantir la précision des mesures
  • Conception d'une base de données performante pour gérer un grand volume de données temporelles
  • Mise en place d'algorithmes d'analyse capables d'identifier des tendances et anomalies
  • Création de visualisations pertinentes et intuitives pour présenter les données
  • Déploiement du système dans différentes conditions environnementales
04

Solutions implémentées

Pour répondre à ces défis, j'ai mis en œuvre plusieurs solutions techniques :

  • Mode veille profonde des ESP32 entre les mesures pour maximiser l'autonomie
  • Protocole MQTT-SN optimisé pour les réseaux contraints avec QoS adapté aux besoins
  • Mise en cache locale sur les capteurs en cas de déconnexion temporaire
  • Algorithmes de calibration avec compensation des facteurs environnementaux
  • Sharding et rétention configurables dans InfluxDB pour gérer la croissance des données
  • Analyses prédictives utilisant des modèles statistiques pour identifier les tendances
  • Tableaux de bord Grafana avec différents niveaux de granularité temporelle
  • Boîtiers étanches et résistants pour protéger les capteurs des intempéries
05

Technologies utilisées

ESP32 Arduino MQTT Mosquitto InfluxDB Grafana Python Pandas NumPy Docker Node-RED Raspberry Pi
06

Résultats et acquis

Ce projet m'a permis de développer des compétences significatives en :

  • Conception et déploiement de systèmes IoT complets
  • Programmation de microcontrôleurs pour des applications de capteurs
  • Implémentation de protocoles de communication efficaces et fiables
  • Gestion de bases de données temporelles à haute performance
  • Techniques d'analyse de données et modélisation statistique
  • Conception d'interfaces de visualisation de données intuitives
  • Déploiement d'applications conteneurisées avec Docker
  • Gestion de projet technique avec contraintes matérielles et logicielles

Le système a été déployé avec succès sur trois sites différents, collectant plus de 500 000 points de données sur une période de quatre semaines. L'analyse a permis d'identifier des microclimats locaux et des corrélations intéressantes entre différentes variables météorologiques. Les tableaux de bord développés ont été présentés lors d'une journée porte ouverte du département, suscitant un intérêt significatif pour cette application concrète des technologies IoT et d'analyse de données.
